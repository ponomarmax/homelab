## Text
Most ML learning paths don’t prepare you for real systems.

So I decided to build one.

Background:
I’ve spent 9+ years building backend systems (.NET, distributed services, high-load environments).

Now I’m transitioning toward AI / ML.

But instead of jumping straight into models, I’m starting from something more fundamental:

→ building a production-like system from scratch

This project combines:
- infrastructure (Docker-based)
- smart home automation (Home Assistant)
- real-world data collection (environment + future physiology data)
- and later — ML experiments

Why?

Because real ML problems are not about models first.
They are about:
- messy data
- system design
- constraints
- and iteration over time

Constraints I’m working with:
- single machine (8GB RAM)
- no Kubernetes
- no overengineering
- everything reproducible

The system will evolve step by step:

1) Infrastructure platform
2) Monitoring & observability
3) Smart home layer
4) Sensor-based data collection
5) External data integrations
6) Analysis
7) ML experiments

Goal:
→ understand how data, systems, and ML actually connect in practice

I’ll be sharing along the way:
- engineering decisions
- trade-offs
- mistakes and fixes
- real progress (not tutorials)

No hype. Just building in public.

If you’ve made a similar transition (backend → ML / AI),
what was the hardest part for you?

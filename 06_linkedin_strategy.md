# LinkedIn Strategy

## Goal

Build visibility for a transition toward AI / ML Engineering through a real engineering project.

---

## Narrative

This project should gradually position the author as:

1. infrastructure-minded engineer
2. smart home / systems engineer
3. data-focused engineer
4. AI / ML engineer in transition with real project work

---

## What to Publish

Focus on:
- engineering decisions
- real problems and fixes
- architectural trade-offs
- visible progress
- lessons learned
- later: data analysis insights

---

## Post Structure

Recommended structure:
1. context
2. problem or decision
3. implementation or resolution
4. takeaway / insight
5. visualization (try to figure out what fits best in this context)

---

## Frequency

2-3 posts per week maximum.
Quality is more important than volume.

---

## What Makes a Post Worth Publishing

A post is worth publishing if it contains at least one of:
- visible checkpoint
- real engineering trade-off
- useful lesson
- meaningful screenshot / diagram / graph
- clear narrative progress

---

## Visual Preference

Preferred visuals:
- architecture diagrams
- screenshots of dashboards
- before/after comparisons
- simplified flow diagrams
- charts based on real collected data

---

## Tone

- clear
- technical but readable
- no hype
- no fake expertise
- no exaggerated AI claims

---

## Main Outcome

The project should create:
- stronger profile positioning
- credible technical signal
- GitHub traffic
- inbound opportunities

# Canonical Contracts V1

## Purpose

This document defines the canonical transport contracts between the collector app and backend services.

These contracts are transport-oriented, not analytical.

Goals:
- keep the outer envelope stable across sensors
- allow sensor-specific payload diversity
- separate ingestion from parsing
- retain raw data first
- keep the design extensible for future sensors

Initial implementation target:
- Polar Verity Sense

Planned extension:
- Muse Athena

---

## Scope

This specification covers:
- SessionContract
- StreamDescriptorContract
- UploadChunkContract
- AckContract
- ErrorContract
- payload registry rules
- versioning rules
- time semantics
- ingestion guarantees

This specification does **not** define:
- normalized analytical schemas
- feature schemas
- visualization schemas
- database physical schema
- ML-ready schemas

---

## Core Principles

1. Stable outer metadata, flexible inner payload
2. Ingestion is not parsing
3. Raw payloads are always retained
4. Sensor-specific logic lives after ingestion
5. One collector app can support multiple sensor adapters
6. Contracts must support both simple scalar streams and multi-channel streams
7. Contracts must remain lightweight and human-inspectable where possible

---

## Contract Model

The transport model consists of:

1. SessionContract
2. StreamDescriptorContract
3. UploadChunkContract
4. AckContract
5. ErrorContract

Payload contracts are versioned separately and referenced by schema id.

Repository schema files live in `packages/schemas/`.
Transport contracts live under `packages/schemas/transport/`.
Sensor-specific payload contracts live under `packages/schemas/payloads/`.
Examples live under `packages/schemas/examples/`.

## 1. SessionContract

### Purpose
Represents one logical recording or collection session.

### JSON shape

```json
{
  "schema_version": "1.0",
  "session_id": "string",
  "device_session_id": "optional string",
  "session_mode": "online_live | offline_recording | file_import | playback",
  "collector": {
    "collector_id": "string",
    "runtime_type": "ios | android | desktop | server_import",
    "app_version": "string",
    "build_version": "optional string"
  },
  "device": {
    "vendor": "polar | muse | other",
    "model": "string",
    "device_id": "string"
  },
  "time": {
    "started_at_source": "timestamp",
    "started_at_server": "optional timestamp"
  },
  "metadata": {
    "notes": "optional string",
    "tags": ["optional", "strings"]
  }
}
Required fields
schema_version
session_id
session_mode
collector.collector_id
collector.runtime_type
collector.app_version
device.vendor
device.model
device.device_id
time.started_at_source
Notes
session_id is generated by the collector and must be globally unique enough for the project.
device_session_id is optional and can be used if the device or SDK has its own session identifier.
started_at_source is the collector-side session start time.
started_at_server is assigned by backend if needed.
2. StreamDescriptorContract
Purpose

Describes one logical stream inside a session.

JSON shape
{
  "schema_version": "1.0",
  "stream_id": "string",
  "session_id": "string",
  "stream_family": "cardio | motion | neural | optical | derived | quality | marker | unknown",
  "stream_type": "hr | ppi | ppg | acc | gyro | mag | eeg | optics | fnirs | marker | unknown",
  "origin": "raw | vendor_processed | server_derived",
  "mode": "online_live | offline_recording | file_import | playback",
  "payload_schema": "string",
  "payload_version": "string",
  "metadata": {
    "sample_rate_hz": "optional number",
    "units": "optional string",
    "channels": ["optional", "channel names"],
    "notes": "optional string"
  }
}
Required fields
schema_version
stream_id
session_id
stream_family
stream_type
origin
mode
payload_schema
payload_version
Notes
stream_id must be unique within a session.
payload_schema identifies the inner payload contract, for example:
polar.hr
polar.ppi
polar.acc
muse.eeg
muse.ppg
payload_version versions the payload independently of the outer transport version.
3. UploadChunkContract
Purpose

Represents one upload unit from collector to backend.

JSON shape
{
  "schema_version": "1.0",
  "chunk_id": "string",
  "session_id": "string",
  "stream_id": "string",
  "sequence": 1,
  "time": {
    "device_time_reference": "optional string",
    "first_sample_received_at_collector": "timestamp",
    "uploaded_at_collector": "timestamp",
    "received_at_server": "optional timestamp"
  },
  "transport": {
    "encoding": "json",
    "compression": "none | gzip",
    "payload_schema": "string",
    "payload_version": "string"
  },
  "payload": {}
}
Required fields
schema_version
chunk_id
session_id
stream_id
sequence
time.first_sample_received_at_collector
time.uploaded_at_collector
transport.encoding
transport.payload_schema
transport.payload_version
payload
Notes
chunk_id must be unique enough for idempotency.
sequence is monotonically increasing within one stream.
device_time_reference is optional because some SDKs provide strong device time and some do not.
received_at_server is assigned by backend when persisted or accepted.
time.first_sample_received_at_collector must equal the first sample timestamp in payload.samples[].received_at_collector when samples exist.
4. AckContract
Purpose

Returned by backend after upload attempt.

JSON shape
{
  "accepted": true,
  "status": "accepted | duplicate | rejected",
  "chunk_id": "string",
  "session_id": "string",
  "stream_id": "string",
  "received_at_server": "timestamp",
  "storage": {
    "raw_persisted": true,
    "storage_path": "optional string"
  },
  "message": "optional string"
}
Required fields
accepted
status
chunk_id
session_id
stream_id
received_at_server
storage.raw_persisted
Rules
accepted=true only if the backend accepted ownership of the chunk.
duplicate means the backend recognized the chunk and did not need to store it again.
rejected means the chunk failed validation or persistence.
5. ErrorContract
Purpose

Returned when the request fails validation or processing.

JSON shape
{
  "accepted": false,
  "status": "rejected",
  "error_code": "validation_error | unsupported_schema | persistence_error | malformed_request",
  "message": "string",
  "details": [
    {
      "field": "string",
      "issue": "string"
    }
  ]
}
Required fields
accepted
status
error_code
message

8. Versioning Rules
Outer contracts
current transport schema version: 1.0
Inner payloads
version independently
payload schema id and payload version are both mandatory
transport changes must not silently break payload compatibility
Compatibility
backward-compatible payload changes should increment minor version
breaking payload changes should increment major version
draft schemas must be clearly marked as draft
9. Time Semantics

Three kinds of time may exist:

source / collector wall-clock time
device-provided time or device reference
server receive time
Current rule
collector timestamps are mandatory
device time is optional
server receive time is backend-assigned
Why

Different devices and SDKs provide different time guarantees.
The transport contract must not assume perfect device clocks.

10. Ingestion Guarantees

The backend ingestion layer must:

validate the outer envelope
store the raw payload durably before any deep parsing
return AckContract or ErrorContract
support duplicate-safe behavior using chunk identity

The backend ingestion layer does not have to:

fully parse sensor-specific payloads immediately
compute features
normalize into analytical structures during initial acceptance

In other words, ingestion validates transport contract only and treats payload as opaque stream-defined JSON.
11. Raw vs Normalized vs Derived
Raw
exact uploaded chunk
immutable
debugging source of truth
Normalized
parsed sensor-specific records
structured for querying and charts
Derived
HRV
movement index
quality flags
epoch summaries
future analytical features
12. Storage Boundary

This contract does not dictate:

SQL vs files vs object storage
table design
partitioning strategy

It only dictates what must be accepted and durably preserved.

13. First Endpoint Recommendation

Recommended first endpoint:

POST /api/v1/ingestion/chunks

Body:

one UploadChunkContract

Future extension:

batch upload endpoint if needed
14. Manual Validation Priority

Before automation grows, the system should support manual validation by:

opening raw files
inspecting stored chunk JSON
matching session_id / stream_id / chunk_id
matching iOS debug state to backend stored data
